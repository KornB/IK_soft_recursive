# Data Format

This document describes the raw sensor log format, processed datasets, and model input/output definitions used in this repository.

---

## 1. Raw CSV format

The preprocessing scripts expect a raw `.csv` file containing:

- `time`
- `p1`, `p2`, `p3`, `p4`, `p5`, `p6`
- `ks`
- pose data for `0A`
- pose data for `0B`
- pose data for `0C`

### Required columns

| Column group | Required columns |
|---|---|
| Time | `time` |
| Pressure | `p1`, `p2`, `p3`, `p4`, `p5`, `p6` |
| Index | `ks` |
| 0A pose | `0A_pos_x`, `0A_pos_y`, `0A_pos_z`, `0A_orient_x`, `0A_orient_y`, `0A_orient_z`, `0A_orient_w` |
| 0B pose | `0B_pos_x`, `0B_pos_y`, `0B_pos_z`, `0B_orient_x`, `0B_orient_y`, `0B_orient_z`, `0B_orient_w` |
| 0C pose | `0C_pos_x`, `0C_pos_y`, `0C_pos_z`, `0C_orient_x`, `0C_orient_y`, `0C_orient_z`, `0C_orient_w` |

### Notes

- keep the raw column names unchanged when using the current preprocessing scripts
- each sensor stream must include both position and quaternion orientation
- keep units consistent across all data files

Recommended units:
- position: mm
- pressure: bar or kPa
- angle: rad or deg

---

## 2. Preprocessing logic

The current preprocessing scripts do not use every raw row directly.  
Instead, they:

1. select rows around pressure-change events
2. apply a sample shift to account for delay
3. estimate zero-reference poses from zero-pressure regions
4. transform raw poses into local segment features
5. save processed datasets and normalization statistics

Typical outputs:
- `.npz` processed training datasets
- `.npz` normalization files
- `.csv` inspection tables

## 3. Proximal pressure model

### Purpose

Predict proximal chamber pressures:

```text
[P1, P2, P3]
```

### Input

```text
[PX, PY, cosP, sinP, dPX, dPY, dcosP, dsinP]
```

### Saved processed CSV format

```text
PX, PY, cosP, sinP, dPX, dPY, dcosP, dsinP, P1, P2, P3
```

### Shapes

```text
X.shape = [N, 8]
Y.shape = [N, 3]
```

---

## 4. Distal pressure model

### Purpose

Predict distal chamber pressures:

```text
[P4, P5, P6]
```

### Input

```text
[PX, PY, cosP, sinP, dPX, dPY, dcosP, dsinP]
```

### Saved processed CSV format

```text
PX, PY, cosP, sinP, dPX, dPY, dcosP, dsinP, P4, P5, P6
```

### Shapes

```text
X.shape = [N, 8]
Y.shape = [N, 3]
```

---

## 5. P2D response model

### Purpose

Predict local distal 2D response from proximal actuation.

### Input

```text
[P1, P2, P3]
```

### Output

```text
[X, Y]
```

### Saved processed CSV format

```text
P1, P2, P3, X, Y
```

### Shapes

```text
X.shape = [N, 3]
Y.shape = [N, 2]
```

---

## 6. D2P response model

### Purpose

Predict local proximal 2D response from distal actuation.

### Recommended format

```text
[P4, P5, P6] -> [X, Y]
```

### Recommended saved processed CSV format

```text
P4, P5, P6, X, Y
```

### Shapes

```text
X.shape = [N, 3]
Y.shape = [N, 2]
```

---

## 7. RSCC state definition

For the downstream control logic, the important state variables are:

```text
s_prox = proximal segment-end tip state
s_dist = distal segment-end tip state
```

These states are converted into the model input features during preprocessing or runtime feature extraction.

---

## 8. Output files generated by preprocessing

Typical output names include:

```text
result<log_name>2.npz
result<log_name>_norm.npz
result<log_name>.csv
```

For P2D-related preprocessing, the current naming may use a `P2Dresult...` prefix.
